📝 HTML ფორმები

ყოვლისმომცველი ინტერაქტიული სწავლების გზამკვლევი - შეისწავლეთ ფორმების შექმნა, ვალიდაცია და მომხმარებელთან ურთიერთობა

შესავალი ფორმის მიმოხილვა Input ელემენტი Textarea ლეიბლები დაჯგუფება რადიო და ჩექბოქსი ჩამოსაშლელი სიები ღილაკები ვალიდაცია ქვიზი

🎯 შესავალი

ეს ყოვლისმომცველი გაკვეთილი განიხილავს HTML5-ში ფორმების კონცეფციას, იკვლევს მათ დანიშნულებას და იძლევა მათი გამოყენების დეტალურ მაგალითებს. ასევე განვიხილავთ ეფექტური, ხელმისაწვდომი ფორმების შექმნისას გამოყენებულ ძირითად ელემენტებს და ატრიბუტებს.

💡 რატომ არის ფორმები მნიშვნელოვანი

თანამედროვე ვებ აპლიკაციებში ფორმები აუცილებელია უკუკავშირის მისაღებად, პირადი ინფორმაციის შესაგროვებლად, გადახდების დასამუშავებლად, მომხმარებლის პრიორიტეტების გამოსაყოფად და მრავალი სხვა მიზნისთვის. ისინი შეიძლება იყოს მარტივი ფორმები, როგორიცაა ავტორიზაციის ფორმა, ან რთული მრავალგვერდიანი რეგისტრაციის პროცესები.

ფორმების დიზაინის ძირითადი პრინციპები

როდესაც თქვენს გვერდებზე ფორმებს ამატებთ, გახსოვდეთ, რომ მომხმარებლები ხშირად ყოყმანობენ საკუთარი მონაცემების გაზიარებაში. დაიცვით ეს მნიშვნელოვანი წესები:

  • ხელმისაწვდომობა: გამოიყენეთ შესაბამისი ფორმის ელემენტები მომხმარებლებს მონაცემების იოლად შესაყვანად
  • მონაცემთა სტრუქტურა: შეაგროვეთ ინფორმაცია შენახვისა და გამოყენებისთვის სასურველ ფორმატში
  • მინიმალიზმი: შეაგროვეთ მხოლოდ ის ინფორმაცია, რაც ნამდვილად გჭირდებათ
  • გასაგები კომუნიკაცია: მიუთითეთ მომხმარებლებს ზუსტად რა მონაცემები მოეთხოვება და რა ფორმატში

🏗️ ფორმის ელემენტის მიმოხილვა

ფორმა არის დოკუმენტის ნაწილი, რომელიც გამოიყენება მომხმარებლების მიერ შეყვანილი ინფორმაციის შესაგროვებლად. <form> ელემენტი გვევლინება კონტეინერად სხვადასხვა ფორმის კონტროლებისთვის.

<form> ფორმის ელემენტები </form>

📝 მნიშვნელოვანი შენიშვნა

ერთ გვერდზე ფორმების რაოდენობა HTML სტანდარტით არ არის შეზღუდული, რაც საშუალებას იძლევა მოქნილ გვერდის დიზაინზე.

ფორმის ატრიბუტები

<form> ელემენტი შეიცავს ინფორმაციას, რომელიც ეხება ფორმას ან მისი ყველა ელემენტს. ვნახოთ ძირითადი ატრიბუტები:

accept-charset

განსაზღვრავს სპეისით გამოყოფილ სიმბოლოების კოდირების სიას ფორმის გაგზავნისთვის.

<form accept-charset="utf-8 iso-8859-1"> <!-- ფორმის შინაარსი --> </form>

action

განსაზღვრავს URL მისამართს, სადაც ფორმის მონაცემები უნდა გაიგზავნოს გაგზავნისას.

<form action="/submit-form" method="POST"> <!-- ფორმის შინაარსი --> </form>

autocomplete

აკონტროლებს თუ არა ბრაუზერი უნდა ავტომატურად შეავსოს ფორმის ველები მომხმარებლის წინა ჩანაწერების საფუძველზე.

<form autocomplete="on"> <!-- ავტოშევსების ჩართვა --> <form autocomplete="off"> <!-- ავტოშევსების გამორთვა -->

method

განსაზღვრავს HTTP მეთოდს ფორმის მონაცემების გაგზავნისას (GET ან POST).

<form method="GET"> <!-- საძებნი ფორმებისთვის, მცირე მონაცემებისთვის --> <form method="POST"> <!-- კონფიდენციალური მონაცემებისთვის, ფაილების ატვირთვისთვის -->

novalidate

Boolean ატრიბუტი, რომელიც ხელს უშლის ბრაუზერს ფორმის ვალიდაციაში გაგზავნისას.

<form novalidate> <!-- ფორმა არ იქნება ვალიდაცირებული ბრაუზერის მიერ --> </form>

enctype

განსაზღვრავს როგორ უნდა იყოს კოდირებული ფორმის მონაცემები სერვერზე გაგზავნისას.

<!-- ფაილების ატვირთვისთვის --> <form enctype="multipart/form-data" method="POST"> <!-- ჩვეულებრივი ფორმებისთვის (ნაგულისხმევი) --> <form enctype="application/x-www-form-urlencoded"> <!-- ტექსტისთვის --> <form enctype="text/plain">

ფორმის ძირითადი სტრუქტურა

საკონტაქტო ფორმის ნიმუში

⌨️ Input ელემენტი

<input> ელემენტი არის ფორმების მთავარი ელემენტი. იგი წარმოადგენს ტიპიზირებულ მონაცემთა ველს ფორმის კონტროლით, რომელიც საშუალებას აძლევს მომხმარებლებს რედაქტირება გაუკეთონ მონაცემებს. ელემენტის გარეგნობა და ქცევა დამოკიდებულია type ატრიბუტის მნიშვნელობაზე.

პოპულარული Input ტიპები

type="text"

ნაგულისხმევი ტიპი, რომელიც საშუალებას აძლევს მომხმარებლებს ტექსტის შეყვანას.

<input type="text" name="username" placeholder="თქვენი სახელი" required>

type="number"

მომხმარებელს აძლევს საშუალებას რიცხვების შეყვანას. შეიძლება შეიცავდეს მინიმუმ და მაქსიმუმ მნიშვნელობებს.

<input type="number" name="age" min="0" max="120" placeholder="თქვენი ასაკი" required>

type="email"

მომხმარებელს აძლევს საშუალებას ელ-ფოსტის მისამართის შეყვანას. ავტომატურად ვალიდაციას ატარებს ელ-ფოსტის ფორმატზე.

<input type="email" name="email" placeholder="თქვენი ელ-ფოსტა" required>

type="password"

მომხმარებელს აძლევს საშუალებას პაროლის შეყვანას, რომელიც დამალულია.

<input type="password" name="password" placeholder="თქვენი პაროლი" required>

type="color"

მომხმარებელს აძლევს საშუალებას ფერის არჩევის.

<input type="color" name="favoriteColor" value="#ff0000">

type="date"

მომხმარებელს აძლევს საშუალებას თარიღის არჩევის.

<input type="date" name="birthdate" required>

type="file"

მომხმარებელს აძლევს საშუალებას ფაილების ატვირთვის.

<input type="file" name="profilePicture" accept="image/*">

Input ელემენტის ატრიბუტები

Input ელემენტს აქვს მრავალი ატრიბუტი, რომელიც აკონტროლებს მის ქცევას და გარეგნობას. აქ არის ყველაზე ხშირად გამოყენებული:

required

მნიშვნელოვანი ატრიბუტი, რომელიც მიუთითებს, რომ ველი აუცილებელია ფორმის გაგზავნისთვის.

<input type="text" name="username" required>

placeholder

მომხმარებელს აძლევს მითითებას, თუ რა უნდა შეიყვანოს ველში.

<input type="text" name="search" placeholder="ძებნა... ">

maxlength

მიზეზდება, რომ მომხმარებელს არ შეეძლოს ველის შევსება მითითებული სიმბოლოების რაოდენობაზე მეტით.

<input type="text" name="username" maxlength="20">
